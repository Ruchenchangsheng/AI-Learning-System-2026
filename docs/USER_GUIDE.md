## 🚀 第一天设置

### 步骤1：创建GitHub仓库
1. 访问 https://github.com
2. 点击右上角"+" → "New repository"
3. 填写信息：
   - Repository name: `AI-Learning-System-2026`
   - Description: `AI学习管理系统 - 小爪智能助手`
   - Public (公开)
   - 勾选"Add a README file"
4. 点击"Create repository"

### 步骤2：本地环境配置
```bash
# 1. 安装Git
sudo apt install git  # Ubuntu/Debian
# 或从 https://git-scm.com/ 下载

# 2. 配置Git
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"

# 3. 克隆仓库
git clone https://github.com/你的用户名/AI-Learning-System-2026.git
cd AI-Learning-System-2026
```

### 步骤3：联系小爪
告诉小爪：
1. 你的GitHub用户名
2. 仓库地址
3. 希望开始的日期

小爪会帮你初始化系统并开始每日服务。

## 📅 每日学习流程

### 早上 (08:00-09:00)
1. **接收计划**：小爪通过QQ发送今日学习计划
2. **查看任务**：打开今日日志文件
   ```bash
   cat progress/daily_logs/$(date '+%Y-%m-%d').md
   ```
3. **确认目标**：明确今日学习重点

### 学习期间 (09:00-18:00)
1. **专注学习**：按计划学习
2. **及时记录**：在日志文件中记录：
   - 学习内容
   - 代码练习
   - 遇到的问题
   - 解决方案
3. **完成任务**：在表格中标记完成情况

### 晚上 (20:00-22:00)
1. **模拟面试**：与小爪进行技术面试
2. **总结反思**：填写日志的总结部分
3. **提交进度**：推送到GitHub
   ```bash
   git add .
   git commit -m "今日学习总结"
   git push origin main
   ```

## 📝 日志填写指南

### 每日日志结构
```
# 日期 学习日志

## 今日计划表
[表格形式，标记完成情况]

## 学习内容记录
[记录具体学习内容]

## 完成情况总结
[统计和评估]

## 收获与思考
[学习感悟]

## 遇到的问题
[困难和解决方案]

## 明日计划
[明天要学什么]
```

### 填写技巧
1. **具体详细**：不要只写"学习了Python"，要写"学习了Python的if-else语句，写了5个练习题"
2. **代码记录**：把练习的代码粘贴到日志中
3. **问题明确**：遇到问题要具体描述，方便小爪帮助
4. **计划具体**：明日计划要具体可执行

## 🔄 GitHub操作指南

### 基本命令
```bash
# 拉取最新更新
git pull origin main

# 查看状态
git status

# 添加文件
git add .

# 提交更改
git commit -m "描述性信息"

# 推送到GitHub
git push origin main

# 查看历史
git log --oneline -10
```

### 常见场景
```bash
# 场景1：忘记提交昨天的进度

git push origin main

# 场景2：冲突解决
git pull origin main
# 手动解决冲突，然后：
git add .
git commit -m "解决冲突"
git push origin main

# 场景3：查看特定文件历史
git log --oneline -- progress/daily_logs/2026-03-29.md
```

## 🛠️ 故障排除

### 常见问题

#### Q1：Git操作失败
```bash
# 检查网络
ping github.com

# 检查Git配置
git config --list

# 重新认证
git remote set-url origin https://你的用户名:你的token@github.com/你的用户名/AI-Learning-System-2026.git
```

#### Q2：日志文件打不开
```bash
# 使用正确编码
cat progress/daily_logs/日期.md

# 或使用文本编辑器
vim progress/daily_logs/日期.md
# 或
code progress/daily_logs/日期.md
```

#### Q3：学习计划不合适
1. 告诉小爪具体问题
2. 提供你的学习感受
3. 小爪会调整计划

#### Q4：时间安排冲突
1. 提前告诉小爪
2. 调整学习时间
3. 补录学习进度

### 紧急情况
1. **系统完全无法使用**：联系小爪紧急处理
2. **数据丢失**：从GitHub恢复
3. **学习中断**：制定恢复计划

## 💡 高效学习技巧

### 时间管理
1. **番茄工作法**：25分钟专注，5分钟休息
2. **时间块**：把大任务分解为小时间块
3. **优先级**：先完成重要且紧急的任务
4. **避免多任务**：一次只做一件事

### 学习策略
1. **主动学习**：不要只看，要动手写代码
2. **费曼技巧**：尝试向别人解释你学到的
3. **间隔重复**：定期复习旧知识
4. **项目驱动**：通过做项目学习

### 心态调整
1. **接受困难**：学习新技术本来就有挑战
2. **庆祝小胜利**：每完成一个任务就鼓励自己
3. **保持耐心**：进步需要时间积累
4. **寻求帮助**：不要卡住超过15分钟
